### 1.2 用户旅程地图（关键场景：遇到税务问题）

```
阶段：发现 → 寻找 → 评估 → 使用 → 验证

触点：申报焦虑 → 群里提问/搜索 → 查看推荐答案 → 采纳建议 → 问题解决

情绪曲线：
😰焦虑 ────→ 🔍寻找 ──→ 🤔评估 ──→ 📝执行 ──→ 😊安心
                  ↑
            没人回答就流失
            没人回答就流失

机会点：
① 主动推送合规提醒（防患于未然）
② 社区里有真实答案（不是广告）
③ 信任体系让好答案浮现
④ 采纳后通知回答者（正向激励）
```

**核心洞察**：用户最痛苦的时刻不是"不知道答案"，而是"知道群里没人能回答"。

## 二、产品战略

### 2.1 价值主张

**一句话定位**：OPC老板的互相帮助平台

**Slogan**：你不是一个人在单打独斗

**北极星指标**：问题解决率 = 有最佳答案且被提问者采纳的问题数 / 总提问数

### 2.2 三级指标体系

| 层级 | 指标 | 第1个月 | 第3个月 | 第1年 |
|------|------|---------|---------|-------|
| 北极星 | 问题解决率 | >60% | >75% | >85% |
| 过程 | 日有效回答数 | >50 | >200 | >500 |
| | 平均回答时间 | <2小时 | <30分钟 | <15分钟 |
| | 回答采纳率 | >40% | >55% | >65% |
| 健康 | DAU/MAU | >20% | >30% | >40% |
| | 7日留存 | >30% | >40% | >50% |

### 3.2 首页：价值密度最高的信息流

- 今日热门问答（已解决 ✅）
- 最新经验分享（踩坑/资源/成果）
- 有人@你的通知摘要
- 关注话题的新回答

### 3.3 问答模块

**发布问题流程**：标题 → 详情 → 标签 → 发布

**回答排序算法**：
```
得分 = 基础分 × 质量因子 × 时间因子
基础分 = 点赞数 + 评论互动×0.5
质量因子 = 最佳答案×3.0 × 作者信誉归一化
时间因子 = e^(-0.1×天数)  [新答案不被永久压底]
```

### 3.4 搭子模块（差异化特色）

```
用户A（技能：开发） + 需要：运营
用户B（技能：运营） + 需要：开发
    ↓
算法匹配 → 推荐配对 → 互相帮助
```

## 五、技术架构

### 5.1 技术栈

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| 前端框架 | Vue3 + Composition API | Alex熟悉，TS好 |
| 状态管理 | Pinia | Vue3官方推荐 |

| 数据库 | PostgreSQL | JSON+全文搜索+数组 |
| 缓存 | Redis | 会话+限流+防刷 |
| 认证 | 微信OAuth 2.0 + JWT | 无密码登录 |
| 实时 | SSE（Server-Sent Events）| 比WebSocket简单 |
| 图片存储 | 腾讯云COS | 帖子图片+头像 |
| 部署 | Docker + Docker Compose | 环境一致 |

### 5.2 数据库设计（核心表）

```sql
-- 用户表
CREATE TABLE users (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    nickname        VARCHAR(50) NOT NULL,
    avatar_url      VARCHAR(500),
    bio             VARCHAR(200),
    city            VARCHAR(50),
    industry        VARCHAR(50),
    opc_years       INTEGER DEFAULT 0,
    credit_score    INTEGER DEFAULT 0,     -- 声望值
    is_verified     BOOLEAN DEFAULT FALSE, -- OPC认证
    created_at      TIMESTAMP DEFAULT NOW()
);

-- 问答表
CREATE TABLE questions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id),
    title           VARCHAR(200) NOT NULL,
    body            TEXT,
    tags            VARCHAR(50)[],
    view_count      INTEGER DEFAULT 0,
    answer_count    INTEGER DEFAULT 0,
    is_solved       BOOLEAN DEFAULT FALSE,
    best_answer_id  UUID,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- 回答表
CREATE TABLE answers (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    question_id     UUID REFERENCES questions(id) ON DELETE CASCADE,
    user_id         UUID REFERENCES users(id),
    body            TEXT NOT NULL,
    upvote_count    INTEGER DEFAULT 0,
    is_best         BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- 帖子表
CREATE TABLE posts (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id),
    post_type       VARCHAR(20) NOT NULL,  -- experience/resource/recruit/rant
    title           VARCHAR(200) NOT NULL,
    body            TEXT,
    images          VARCHAR(500)[],
    tags            VARCHAR(50)[],
    upvote_count    INTEGER DEFAULT 0,
    comment_count   INTEGER DEFAULT 0,
    favorite_count  INTEGER DEFAULT 0,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- 点赞表（通用）
CREATE TABLE likes (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id) ON DELETE CASCADE,
    target_type     VARCHAR(20) NOT NULL,   -- post/answer
    target_id       UUID NOT NULL,
    created_at      TIMESTAMP DEFAULT NOW(),
    UNIQUE(user_id, target_type, target_id)
);

-- OPC认证表
CREATE TABLE business_verification (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID UNIQUE REFERENCES users(id),
    company_name        VARCHAR(200) NOT NULL,
    unified_credit_code VARCHAR(50) NOT NULL,
    license_front       VARCHAR(500) NOT NULL,
    status              VARCHAR(20) DEFAULT 'pending',
    created_at          TIMESTAMP DEFAULT NOW()
);

-- 通知表
CREATE TABLE notifications (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id) ON DELETE CASCADE,
    type            VARCHAR(30) NOT NULL,
    title           VARCHAR(200),
    body            TEXT,
    target_type     VARCHAR(20),
    target_id       UUID,
    is_read         BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- 声望值等级索引
CREATE INDEX idx_users_credit ON users(credit_score DESC);
CREATE INDEX idx_questions_created ON questions(created_at DESC);
CREATE INDEX idx_answers_question ON answers(question_id);
CREATE INDEX idx_posts_created ON posts(created_at DESC);
-- 全文搜索索引
CREATE INDEX idx_questions_fts ON questions
    USING gin(to_tsvector('simple', title || ' ' || COALESCE(body, '')));
```

## 六、声望值系统

### 6.1 计算规则

| 行为 | 声望 | 每日上限 |
|------|------|---------|
| 发布回答被点赞 | +10/赞 | 50次 |
| 发布帖子被点赞 | +5/赞 | 30次 |
| 回答被采纳为最佳答案 | +50 | 无 |
| 回答被评论 | +2/评论 | 20次 |
| 发布广告（核实） | -100 | 无 |

### 6.2 等级体系

| 等级 | 声望 | 标签 | 权限 |
|------|------|------|------|
| 1 | 0-99 | 新人 | 浏览/提问 |
| 2 | 100-499 | 活跃 | 回答/发帖 |
| 3 | 500-1999 | ⭐社区达人 | 搭子/置顶 |
| 4 | 2000-4999 | ⭐⭐认证专家 | 橙色徽章 |
| 5 | 5000+ | 🏅社区MVP | 金色徽章 |
