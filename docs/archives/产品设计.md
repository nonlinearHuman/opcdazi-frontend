## 一、设计理念

### 1.1 产品核心感受

**关键词**：被看见、被陪伴、被保护

> 不是冰冷的企业服务，而是"懂你一个人打拼"的温度

**情绪板**：
- 孤独感 → 社区互动 → 找到同类
- 恐惧感 → 合规工具 → 安心睡觉
- 迷茫感 → 内容指南 → 明确方向

### 1.2 设计原则

| 原则 | 具体含义 |
|------|---------|
| **温暖但专业** | 不像政府网站那样冰冷，也不像微商那样随意 |
| **工具优先** | 每个功能都要有实际用途，不做花瓶功能 |
| **降低认知负担** | "一个人"已经够累了，产品不能让用户动脑筋 |
| **及时陪伴** | 用户在恐惧/迷茫的时候，第一时间出现 |

### 3.2 合规日历（小程序 + 服务号提醒）

> "让OPC老板安心睡觉，不再担心错过截止日"

- 每年6月30日：上一年度工商年报截止
- 季度末：季度税务申报
- 每月15日：个人所得税申报
- 注册周年：认缴资本到账检查

**服务号提醒机制**：

| 时间节点 | 提醒内容 | 模板消息 |
|---------|---------|---------|
| 截止前30天 | 首个提醒 | 「亲，你的年报还有30天，别忘了哦～」 |
| 截止前7天 | 再次提醒 | 「⚠️ 年报截止还剩7天！」 |
| 截止前1天 | 最后通牒 | 「🔥 明天截止！点击马上处理」 |
| 截止当天 | 紧急提醒 | 「今天！年报截止！再不处理公司要吊销！」 |

**用户输入**：
- 注册时间（自动算出年报截止）
- 税务类型（小规模/一般纳税人）
- 认缴资本金额和期限

### 3.5 种子社群设计

**社群定位**：
> 不是灌水群，而是"OPC老板的深夜茶话会"

**入群门槛**：
- 关注公众号 + 完成注册自检（筛选真实用户）
- 入群前填写表单：行业/注册时间/最关心的问题
- 限额200人（保证质量）

**社群规则**：
1. 禁止广告（违者秒踢）
2. 禁止拉人（防止微商入侵）
3. 鼓励分享真实经验和踩坑故事
4. 管理员每日精选3条有价值内容置顶

**社群内容节奏**：

| 时间 | 内容类型 |
|------|---------|
| 早8点 | 今日合规提醒（机器人） |
| 午12点 | 行业资讯/政策解读 |
| 晚9点 | 话题讨论/经验分享 |
| 随时 | 问答互动（管理员/CA入驻） |

**社群差异化**：
- 每个入群的人都有"OPC老板"身份认同
- 定期分享：真实OPC的赚钱/踩坑故事
- 每月1次：OPC创始人线上meetup（视频会议）

## 四、增长策略

### 4.1 冷启动（0→500用户）

**渠道1：知乎（SEO长尾词）**
- 问题：「一个人开公司要注意什么」「OPC是什么」
- 回答：专业但有温度，文末引导「我建了个OPC群」
- 目标：每天1篇，持续1个月

**渠道2：微信搜一搜**
- 优化公众号名称：「OPC帮」「一人公司」
- 用户搜索「怎么注册一人公司」→ 找到你

**渠道3：小红书**
- 标题：「我一个人开公司，第1年赚了XX万」
- 内容：真实OPC故事，不发广告
- 引导：评论区互动

### 4.2 口碑裂变（500→2000用户）

**裂变机制**：
- 用户分享「合规日历」给朋友 → 双方获得「合规资料包」
- 用户邀请3个OPC老板入群 → 解锁「年度付费社群」折扣

### 4.3 服务商合作（变现）

**合作模式**：
- 代理记账：社区推荐 → 成交返佣20-30%
- 公司注册：社区推荐 → 成交返佣¥500-1500
- 银行开户：社区推荐 → 合作银行返佣
